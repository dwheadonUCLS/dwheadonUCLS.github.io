Python Cheatsheet

The Python syntax found here will be used continually throughout this course. It should be committed to memory. If you have to constantly be referring back to it, the assignments will take you twice as long to do and you will lose precious time. Even though this cheatsheet will always be available to you (except for quizzes and tests), it would be good to maintain your own version (written out) in your notes for easier access and to help you memorize the contents.

[From Module 1: Write your first program]
Create a string value
"characters enclosed in either single or double quotes" "hello"
The string will end with whichever kind of quote you begin with so make sure your enclosed characters don't contain that kind of quote.
Call a function
function_name(parameter1_value, parameter2_value) print("hello")
Not all functions take parameters but you still need the parenthesis. Some functions take multiple parameters: you separate them with a comma
Create a variable or give an existing one a new value
Variables allow us to "remember" or "store" a value so that we can use it later in our program.
variable_name = initial_value pi = 3.14
Note: the variable must be on the left side
Naming rules for variables
  1. Cannot be a reserved keyword
  2. Cannot start with a number
  3. No punctuation, delimeters, or special characters
  4. No spaces
Recommended to use snake case
Use Snake Case
put_underscore_between_words
Use when appropriate: when the purpose of the variable can't be summarized in a single word

[From Module 3: Asking questions and making decisions]
Call a function that returns a value
Some functions return a value. There's no point to calling these kinds of functions unless you actually do something with the results. Usually this will involve saving the value in a variable which you can then use later in your program. The function call will get replaced by the value that it returns.
variable_name = function_name(required, parameters) name = input("What's your name: ")
This is really just the combination of two syntax rules we've already seen (creating variables and calling functions)
f-strings
f-strings are used almost exclusively when you want to print something out that includes the values of some (one or more) varaibles formatted a certain way.
print(f'words you want to print {variable_name} other words you want to print')
There are all kinds of fancy ways to format the values of the variables (e.g. you're printing money and you only want 2 decimal places). See the documentation for more details.
Decisions with two options
if boolean_expression: # Code to run when boolean_expression is True else: # Code to run when boolean_expression is False if age >= 18: print("You're legally an adult'") else: print("You're just a little kid")
The boolean_expression must be something that evaluates to True or False. If it doesn't (e.g 3 + 5) it will produce a TypeError.
The else part is optional but the if part is NOT optional (you can't have an else without an if)
Also note that the else does NOT have a condition. Its condition is implicitly the opposite of the if's. If you try to put a condition after an else it will produce a syntax error.

[From Module 5: More decisions]
Decisions with multiple options
if boolean_expression_1: # Code to run when boolean_expression_1 is True elif boolean_expression_2: # Code to run when boolean_expression_1 is False # AND boolean_expression_2 is True else: # Code to run only when ALL previous boolean_expressions are False if age >= 20: print("You're old'") elif age > 13: print("You're a teenager") else: print("You're just a little kid")
The boolean_expressions must evaluate to True or False. If they don't (e.g 3 + 5) it will produce a TypeError.
The else and elif parts are optional but the if part is NOT optional (you can't have an else without an if)
Also note that the else does NOT have a condition. Its condition is implicitly the opposite of the if's. If you try to put a condition after an else it will produce a syntax error.
Boolean expression with strings
"some string" in "some other string" "cal" in "supercalifragalistic"
This evaluates to True or False and can therefore be used in a conditional (like an if-statement)
Call a method on a "thing"
Methods are like functions except that you have to call them on a thing. The thing could be an actual value or a variable that represents that value.
thing.methodName(required, parameter, values) "hello".upper() word = "hello" word.upper()
Note: not all "things" have methods that can be called on them. It depends on the type of the thing. For strings, the methods available can be found in the str documentation.
Note: in many cases the method won't actually change the value. In such cases, it will return a value which you have to use somehow (e.g. print, save in a variable).
[From Module 6: Loopy loops]
for loop
For going through each of the items in a sequence
for item_name in sequence: # Code to run for each item in the sequence for letter in "bunch of letters": print(letter)
The item_name can be used to refer to the item in the sequence that you're currently on
Any code inside (indented) under the for loop will be run multiple times. Code outside the loop (unindented before or after) will only be run once.
Execute code a specific number of times
for number_name in range(parameters): # Code to run for each number in the sequence # Prints out the numbers from 0 to 99 for num in range(100): print(num)
With 1 parameter, range(num1) produces a sequence of numbers from 0 up to but not including num1
With 2 parameters, range(num1, num2) produces a sequence of numbers from num1 up to but not including num2
With 3 parameters, range(num1, num2, num3) produces a sequence of numbers from num1 up to but not including num2 going up (or down if negative) by num3
[From Module 7: Data representation and indexing]
Get an item at a specific position in a sequence
For strings, this gives you the specific character at that position
string[index] word = "hello" word[0] # gives us "h" word[4] # gives us "o" word[len(word) - 1] # also gives us "o" word[-1] # also gives us "o"
Note: indecies are zero-based
Note: if the index used is bigger than the size of the string, you will get an IndexError
Note: By itself, the bracket notation doesn't do anything. You will want to remember the value that you get (so you can use it later) or do something with it immediately (e.g. print it out)

Create your own function
def function_name(param_name_1, param_name_2): # Code to run (indented) when the method is called # Most functions will want to return something # Parameter values are refered to using the name given above def times_2(num): return num * 2

Note: you call your own functions in the same way as you call the built-in ones (e.g. print, input, etc.)
[From Course 2, Module 1: Review]
Create a list with pre-determined values
[value0, value1, value2, etc] primes = [1, 3, 5, 7, 11, 13]
Get the size of a list
len(list_name) len(primes)
By itself, this doesn't do anything. It's just a value that you can use in another statement.
Get a value from a list
list_name[index] print(primes[3])
Note: this is similar to getting a character from within a string and the same rules apply
Set the value of an element in a list
list_name[index] = new_value primes[3] = 47
Note: we couldn't do this with strings because strings are immutable (unchangeable) but lists are mutable
Add a new element to the end of a list
list_name += [new_value] primes += [103]
Note: there are other ways to change lists using the various list methods
for-each loop
Go through the contents of a list
for variable_name in list: # do something with each of the items in the list # anywhere you use variable_name here it will refer # to the current value in the list numbers = [1, 2, 3, 4, 5] # prints the square of each number in the list... for num in numbers: print(num * num)
Note: this is also similar to strings but unlike strings we're going through the items, not characters
Execute code as long as a condition is True
while loopCondition: # Code to run as long as the # loop condition is true # Keep squaring until you get above 1000000 num = 2 while num < 1000000: num = num * num print(f"Final num {num}")
This kind of loop is usefull when you don't know how many times it will run.
[From Course 2, Module 2: Dictionaries]
Create a dictionary with pre-determined values
{key1: value1, key2: value2} sides = {'triangle': 3, 'square': 4, 'rectangle': 4}
Get a value associated with a key in a dictionary
dict_name[some_key] print(sides['square'])
Set the value associated with a key in a dictionary
dict_name[some_key] = some_value sides['pentagon'] = 5
The key could already be in the dictionary (in which case, the existing value will be replaced) or it could be new to the dictionary (in which case, it is added)
Check if a key is in a dictionary
some_key in dict_name 'square' in sides
By itself, this doesn't do anything. It's just a value that you can use in an if-statement. Oftentimes you will want to check if a key is already in a dictionary before attempting to update its value.
Go through a dictionary
When you go-through a dictionary, it goes through the keys:
for key_variable_name in some_dict: # do something with each of the keys in the dictionary # anywhere you use key_variable_name here it will refer # to the current key in the dictionary for shape in sides: print(f"A {shape} has {sides[shape]} sides")
