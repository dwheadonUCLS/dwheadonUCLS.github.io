Python Cheatsheet

The Python syntax found here will be used continually throughout this course. It should be committed to memory. If you have to constantly be referring back to it, the assignments will take you twice as long to do and you will lose precious time. Even though this cheatsheet will always be available to you (except for quizzes and tests), it would be good to maintain your own version (written out) in your notes for easier access and to help you memorize the contents. Extra credit will be given for maintaining your own handwritten version.

[From Talking to your computer]
Create a variable or give an existing one a new value
Variables allow us to "remember" or "store" a value so that we can use it later in our program.
variable_name = initial_value pi = 3.14
Note: the variable must be on the left side
Naming rules for variables
  1. Cannot be a reserved keyword
  2. Cannot start with a number
  3. No punctuation, delimeters, or special characters
  4. No spaces
Recommended to use snake case
Use Snake Case
put_underscore_between_words
Use when appropriate: when the purpose of the variable can't be summarized in a single word
Call a function
function_name(parameter1_value, parameter2_value) print("hello")
Not all functions take parameters but you still need the parenthesis. Some functions take multiple parameters: you separate them with a comma
Call a function that returns a value
Some functions return a value. There's no point to calling these kinds of functions unless you actually do something with the results. Usually this will involve saving the value in a variable which you can then use later in your program. The function call will get replaced by the value that it returns.
variable_name = function_name(parameter1_value, parameter2_value)
Example: calling a function that returns a value (the letters that the user typed) and saves it in a variable (so we can use it later)
name = input("What's your name: ")

The type of the value / variable will determine the types of operations that you can perform on it. You cannot, for example, divide a string. What would "hello" / 10 be? That doesn't make sense.
Even if it only contains numerals, if it's in quotes it's still a string: "20" / 10 won't work either because it's the string "two zero" not the number twenty.
Built-in functions
There are a bunch of built-in functions that you learn here but the "syntax" for all functions is the same (see "Call a function..." sections above) so they won't be covered here. See the documentation here for more details on these functions (input, int, len, print, str) and others that are available. Note that many of these return a value so if you don't save it in a variable it does nothing.
Example: pointless function call (second line below)
numerals = input("Type a number: ") int(numerals)
The variable numerals is still just a bunch of letters because you didn't do anything with the result of the function int()
Example: purposeful function call
numerals = input("Type a number: ") number = int(numerals)
[From Making decisions]
Conditionally execute code
if boolean_expression_1: # Code to run when boolean_expression_1 is True elif boolean_expression_2: # Code to run when boolean_expression_1 is False # AND boolean_expression_2 is True else: # Code to run when BOTH conditions are False if age >= 20: print("You're old'") elif age > 13: print("You're a teenager") else: print("You're a little kid")
The else and the elif parts are optional but the if part is NOT optional
Also note that the else does NOT have a condition. Its condition is implicitly the opposite of the if's. If you put a condition it will produce a syntax error

[From Manipulating strings]
The in operator
This is a special operator that can only be used on two strings (or variables that hold strings)
string1 in string2 "hello" in "hello world"
Note: This doesn't actually do anything but returns true / false which can be used in an if statement (for example)
Call a method on a "thing"
Methods are like functions except that you have to call them on a thing. The thing could be an actual value or a variable that represents that value.
thing.methodName(required, parameter, values) "hello".upper() word = "hello" word.upper()
Note: not all "things" have methods that can be called on them. It depends on the type of the thing.
Note: in many cases the method won't actually change the value. In such cases, it will return a value which you have to use somehow (e.g. print, save in a variable).
Get an item at a specific position in a sequence
For strings, this gives you the specific character at that position
string[index] word = "hello" word[0] # gives us "h" word[4] # gives us "o" word[len(word) - 1] # also gives us "o" word[-1] # also gives us "o"
Note: indecies are zero-based
Note: if the index used is bigger than the size of the string, you will get an IndexError
Note: By itself, the bracket notation doesn't do anything. You will want to remember the value that you get (e.g. print it or use it in some way)
[From Repeating things]
for loop
for itemName in sequence: # Code to run for each item in the sequence for letter in "bunch of letters": print(letter)
Execute code a specific number of times
for numberName in range(parameters): # Code to run for each number in the sequence # Prints out the numbers from 0 to 99 for num in range(100): print(num)
With 1 parameter, range(num1) produces a sequence of numbers from 0 up to but not including num1
With 2 parameters, range(num1, num2) produces a sequence of numbers from num1 up to but not including num2
With 3 parameters, range(num1, num2, num3) produces a sequence of numbers from num1 up to but not including num2 going up (or down if negative) by num3
Slicing syntax for sequences (including strings)
# Between 2 indecies sequence[startIndex:endIndex] # Up to an index sequence[startIndex:] # After an index sequence[:endIndex] letters = "abcdefgh" letters[2:6] # gives us "cdef" letters[:4] # gives us "abcd" word[4:] # gives us "efgh"
Note: indecies are zero-based
Note: if the index used is bigger than the size of the string, you will get an IndexError
Note: By itself, the bracket notation doesn't do anything. You will want to remember it, print it or use it in some way
[From Storing lists of values]
Create a list with pre-determined values
[value0, value1, value2, etc] primes = [1, 3, 5, 7, 11, 13]
Get the size of a list
len(list_variable) len(primes)
By itself, this doesn't do anything. It's just a value that you can use in another statement.
Set the value of an element in a list
list_name[index] = new_value nums[3] = 10
Note: if the index used is bigger than the size of the list, you will get an IndexError
for-each loop
Go through the contents of a list
for variable_name in list: # do something with each of the items in the list # anywhere you use variable_name here it will refer # to the current value in the list numbers = [1, 2, 3, 4, 5, ...] # prints the square of each number in the list... for num in numbers: print(num * num)
[From Looping in the unknown]
Execute code as long as a condition is True
while loopCondition : # Code to run as long as the # loop condition is True password = None while password != "abc123": password = input("What's the password: ")
Use this kind of loop in situations where you don't know how many times the loop will have to run. In this case, we don't know how many times the password will be entered incorrectly before the user will enter the correct password (ie. sabc123). They may enter it correctly on the first try or it may take them 20 tries to enter the correct password.
